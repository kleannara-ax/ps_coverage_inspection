MES ↔ JRI API 명세서

Jumbo Roll Share Inspector — Manufacturing Execution System Integration Specification

버전v1.0 작성일2026-03-17 시스템JRI (지분 검사 시스템) Base URLhttps://jri.company.com

목차

  1. 검사 실행 (MES → JRI) — 브라우저 URL 호출로 검사 화면 오픈
  2. 검사 결과 전송 (JRI → MES) — 저장 시 자동 POST 전송
  3. 검사 결과 조회 (MES → JRI) — 브라우저 URL 호출로 이력 조회
1 검사 실행 GET
MES → JRI

MES에서 브라우저로 아래 URL을 호출하면 JRI 검사 화면이 열리고 메타데이터가 자동 입력됩니다.

Endpoint
GET {JRI_BASE_URL}/?IND_BCD={값}&LOT_NO={값}&MATNR={값}&...
Request Parameters (Query String)
ParameterTypeDescription
IND_BCD 필수string개별바코드
LOT_NO 선택stringLOT 번호
MATNR 선택string자재코드
MATNR_NM 선택string자재명 (한글·특수문자 URL 인코딩 필요)
USERID 선택string검사자 ID
USERNM 선택string검사자명 (한글 URL 인코딩 필요)
Example
GET https://jri.company.com/?IND_BCD=26228J0039 &LOT_NO=6228J30035 &MATNR=H3SM1240-00000000 &MATNR_NM=KSC%20(FSC)%20240GSM%200000*0000%20%EC%A0%90%EB%B3%B4%EB%A1%A4%20(3) &USERID=USER01 &USERNM=%ED%99%8D%EA%B8%B8%EB%8F%99
Response

브라우저에서 JRI 검사 화면(HTML)이 렌더링됩니다. API 응답이 아닌 웹 페이지입니다.

참고: 검사 순번(차수)은 JRI가 동일 바코드 기준으로 DB에서 자동 채번합니다.
MES에서 순번을 별도로 전달할 필요가 없습니다. (1차, 2차, 3차... 자동 증가)
인코딩: 자재명(MATNR_NM), 검사자명(USERNM) 등 한글·특수문자가 포함된 값은 반드시 URL 인코딩(Percent-encoding)을 적용해야 합니다.
예) 홍길동 → %ED%99%8D%EA%B8%B8%EB%8F%99
2 검사 결과 전송 POST
JRI → MES

사용자가 JRI에서 [검사 결과 저장] 버튼을 클릭하면, DB 저장 완료 후 MES에 자동으로 결과가 전송됩니다.

⚡ 트리거: [검사 결과 저장] 버튼 클릭 → DB 저장 성공 → 자동 MES 전송
Endpoint (MES가 제공해야 할 API)
POST {MES_ENDPOINT_URL}
Request Headers
HeaderValue
Content-Typeapplication/json
Request Body
ParameterTypeDescription
IND_BCD 필수string개별바코드
ResultData 필수number커버리지 결과 값 (ppm 단위)
Request Example
POST {MES_ENDPOINT_URL} Content-Type: application/json { "IND_BCD": "26228J0039", "ResultData": 265.30 }
ResultData 값 설명
값 범위단위설명
0 이상ppm커버리지 비율 × 10,000
coverageRatio = 0.0265 → ResultData = 265.0
0ppm지분이 검출되지 않은 경우
Expected Response (MES가 반환)
FieldTypeDescription
successboolean수신 성공 여부
messagestring결과 메시지
Response Example
// 성공 { "success": true, "message": "수신 완료" } // 실패 { "success": false, "message": "바코드를 찾을 수 없습니다." }
Status Codes
CodeDescription
200 OK전송 성공 (success: true)
400 Bad Request필수 파라미터 누락 (IND_BCD 또는 ResultData)
500 Internal Server ErrorMES 서버 내부 오류
MES 개발자 할 일: 위 규격에 맞는 POST 엔드포인트를 구현하고,
해당 URL을 JRI 담당자에게 전달하면 application.yml에 등록합니다.
# application.yml jri: mes: endpoint-url: http://mes-server.company.com/api/inspection/result
3 검사 결과 조회 GET
MES → JRI

MES에서 브라우저로 아래 URL을 호출하면 해당 바코드의 검사 이력 화면으로 자동 이동합니다.

Endpoint
GET {JRI_BASE_URL}/?tab={뷰타입}&search={개별바코드}
Request Parameters (Query String)
ParameterTypeDescription
tab 필수string history — 카드 뷰 (이미지 미리보기 포함)
history-table — 테이블 뷰 (42개 전체 필드, Excel 내보내기 가능)
search 필수string검색할 개별바코드 값
Example
카드 뷰 (요약 + 이미지) GET https://jri.company.com/?tab=history&search=26228J0039 테이블 뷰 (전체 데이터 + Excel) GET https://jri.company.com/?tab=history-table&search=26228J0039
Response

브라우저에서 검사 이력 화면(HTML)이 렌더링됩니다. API 응답이 아닌 웹 페이지입니다.

뷰 타입별 표시 내용
tab 값뷰 이름표시 내용
history카드 뷰바코드, LOT, 자재코드, 커버리지 요약 카드 + 원본/결과 이미지 미리보기
history-table테이블 뷰42개 전체 필드 테이블 + Excel 내보내기 버튼 + 삭제 기능
조회 결과: 해당 개별바코드의 검사 이력만 필터링되어 표시됩니다.
동일 바코드로 여러 번 검사한 경우 1차, 2차, 3차... 모두 조회됩니다.
